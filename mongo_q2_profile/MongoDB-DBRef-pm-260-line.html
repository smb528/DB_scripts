146=head1 NAME
147
148MongoDB::DBRef - A MongoDB database reference
149
150=head1 VERSION
151
152version v1.4.2
153
154=head1 SYNOPSIS
155
156 my $dbref = MongoDB::DBRef->new(
157 ref => 'my_collection',
158 id => 123
159 );
160
161 $coll->insert( { foo => 'bar', other_doc => $dbref } );
162
163=head1 DESCRIPTION
164
165This module provides support for database references (DBRefs) in the Perl
166MongoDB driver. A DBRef is a special embedded document which points to
167another document in the database. DBRefs are not the same as foreign keys
168and do not provide any referential integrity or constraint checking. For example,
169a DBRef may point to a document that no longer exists (or never existed.)
170
171Generally, these are not recommended and "manual references" are preferred.
172
173See L<Database references|http://docs.mongodb.org/manual/reference/database-references/>
174en the MongoDB manual for more information.
175
176=head1 ATTRIBUTES
177
178=head2 id
179
180Required. The C<_id> value of the referenced document. If the
181C<_id> is an ObjectID, then you must use a L<MongoDB::OID> object.
182
183This may also be specified in the constructor as C<'$id'>.
184
185=head2 ref
186
187Required. The collection in which the referenced document lives. Either a
188L<MongoDB::Collection> object or a string containing the collection name. The
189object will be coerced to string form.
190
191This may also be specified in the constructor as C<'$ref'>.
192
193=head2 db
194
195Optional. The database in which the referenced document lives. Either a
196L<MongoDB::Database> object or a string containing the database name. The
197object will be coerced to string form.
198
199Not all other language drivers support the C<$db> field, so using this
200field is not recommended.
201
202This may also be specified in the constructor as C<'$db'>.
203
204=head2 extra
205
206Optional. A hash reference of additional fields in the DBRef document.
207Not all MongoDB drivers support this feature and you B<should not> rely on
208it. This attribute exists solely to ensure DBRefs generated by drivers that
209do allow extra fields will round-trip correctly.
210
211B<USE OF THIS FIELD FOR NEW DBREFS IS NOT RECOMMENDED.>
