304
305=head1 VERSION
306

309=head1 SYNOPSIS
310
311 use MongoDB::Error;
312 MongoDB::Error->throw("a generic error");
313 MongoDB::DatabaseError->throw(
314 message => $string,
315 result => $hashref,
316 );

318=head1 DESCRIPTION
319
320This class defines a heirarchy of exception objects.
321
322=head1 USAGE
323
324Unless otherwise explictly documented, all driver methods throw exceptions if
325an error occurs.
326
327To catch and handle errors, the L<Try::Tiny> and L<Safe::Isa> modules
328are recommended:
329
330 use Try::Tiny;
331 use Safe::Isa; # provides $_isa
332
333 try {
334 $coll->insert( $doc )
335 }
336 catch {
337 if ( $_->$_isa("MongoDB::DuplicateKeyError" ) ) {
338 ...
339 }
340 else {
341 ...
342 }
343 };
344
345To retry failures automatically, consider using L<Try::Tiny::Retry>.

403=head2 MongoDB::AuthError
404
405This error indicates a problem with authentication, either in the underlying
406mechanism or a problem authenticating with the server.
407
408=head2 MongoDB::ConnectionError
409
410Errors related to network connections.
411
412=head3 MongoDB::HandshakeError
413
414This error is thrown when a connection has been made, but SSL or authentication
415handshakes fail.
416
417=head3 MongoDB::NetworkError
418
419This error is thrown when a socket error occurs, when the wrong number of bytes
420are read, or other wire-related errors occur.
421
422=head2 MongoDB::CursorNotFoundError
423
424This error indicates that a cursor timed out on a server.
425
426=head2 MongoDB::DatabaseError
427
428Errors related to database operations. Specifically, when an error of this type
429occurs, the driver has received an error condition from the server.
430
431Attributes include:

442
443=back
444
445=head3 MongoDB::DuplicateKeyError
446
447This error indicates that a write attempted to create a document with a
448duplicate key in a collection with a unique index. The C<result> attribute is
449a result object.
450
451=head3 MongoDB::NotMasterError
452
453This error indicates that a write or other state-modifying operation was
454attempted on a server that was not a primary. The C<result> attribute is
455a L<MongoDB::CommandResult> object.
456
457=head3 MongoDB::WriteError
458
459Errors indicating failure of a write command. The C<result> attribute is
460a result object.
461
462=head3 MongoDB::WriteConcernError
463
464Errors indicating failure of a write concern. The C<result> attribute is a
465result object.
466
467=head2 MongoDB::DecodingError
468
469This error indicates a problem during BSON decoding; it wraps
470the error provided by the underlying BSON encoder. Note: Encoding errors
471will be thrown as a L</MongoDB::DocumentError>.
472
473=head2 MongoDB::DocumentError
474
475This error indicates a problem with a document to be inserted or replaced into
476the database, or used as an update document.
477
478Attributes include:

483
484document â the document that caused the error
485
486=back
487
488=head2 MongoDB::GridFSError
489
490Errors related to GridFS operations, such a corrupted file.
491
492=head2 MongoDB::InternalError
493
494Errors that indicate problems in the driver itself, typically when something
495unexpected is detected. These should be reported as potential bugs.
496
497=head2 MongoDB::ProtocolError
498
499Errors related to the MongoDB wire protocol, typically problems parsing a
500database response packet.
501
502=head2 MongoDB::SelectionError
503
504When server selection fails for a given operation, this is thrown. For example,
505attempting a write when no primary is available or reading with a specific mode
506and tag set and no servers match.
507
508=head2 MongoDB::TimeoutError
509
510These errors indicate a user-specified timeout has been exceeded.
511
512=head3 MongoDB::ExecutionTimeout
513
514This error is thrown when a query or command fails because C<max_time_ms> has
515been reached. The C<result> attribute is a L<MongoDB::CommandResult> object.
516
517=head3 MongoDB::NetworkTimeout
518
519This error is thrown when a network operation exceeds a timeout, typically
520C<connect_timeout_ms> or C<socket_timeout_ms>.
521

523
