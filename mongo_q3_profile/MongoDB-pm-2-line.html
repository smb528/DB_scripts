120=head1 DESCRIPTION
121
122This is the official Perl driver for L<MongoDB|http://www.mongodb.com>.
123MongoDB is an open-source document database that provides high performance,
124high availability, and easy scalability.
125
126A MongoDB server (or multi-server deployment) hosts a number of databases. A
127database holds a set of collections. A collection holds a set of documents. A
128document is a set of key-value pairs. Documents have dynamic schema. Using dynamic
129schema means that documents in the same collection do not need to have the same
130set of fields or structure, and common fields in a collection's documents may
131hold different types of data.
132

168=head1 USAGE
169
170The MongoDB driver is organized into a set of classes representing different
171levels of abstraction and functionality.
172
173As a user, you first create and configure a L<MongoDB::MongoClient> object to
174connect to a MongoDB deployment. From that client object, you can get
175a L<MongoDB::Database> object for interacting with a specific database.
176
177From a database object, you can get a L<MongoDB::Collection> object for CRUD
178operations on that specific collection, or a L<MongoDB::GridFS> object for
179working with an abstract file system hosted on the database. Each of those
180classes may return other objects for specific features or functions.
181
182See the documentation of those classes for more details or the
183L<MongoDB Perl Driver Tutorial|MongoDB::Tutorial> for an example.
184
185=head2 Error handling
186
187Unless otherwise documented, errors result in fatal exceptions. See
188L<MongoDB::Error> for a list of exception classes and error code
189constants.

191=head1 METHODS

193=head2 connect
194
195 $client = MongoDB->connect(); # localhost, port 27107
196 $client = MongoDB->connect($host_uri);
197 $client = MongoDB->connect($host_uri, $options);
198
199This function returns a L<MongoDB::MongoClient> object. The first parameter is
200used as the C<host> argument and must be a host name or L<connection string
201URI|MongoDB::MongoClient/CONNECTION STRING URI>. The second argument is
202optional. If provided, it must be a hash reference of constructor arguments
203for L<MongoDB::MongoClient::new|MongoDB::MongoClient/ATTRIBUTES>.
204
205If an error occurs, a L<MongoDB::Error> object will be thrown.
206
207B<NOTE>: To connect to a replica set, a replica set name must be provided.
208For example, if the set name is "setA":
209
210 $client = MongoDB->connect("mongodb://example.com/?replicaSet=setA");

216=head1 SEMANTIC VERSIONING SCHEME
217
218Starting with MongoDB C<v1.0.0>, the driver reverts to the more familiar
219three-part version-tuple numbering scheme used by both Perl and MongoDB:
220C<vX.Y.Z>
221
222=over 4
223
224=item *
225
226C<X> will be incremented for incompatible API changes.
227
228=item *
229
230Even-value increments of C<Y> indicate stable releases with new functionality. C<Z> will be incremented for bug fixes.
231
232=item *
233
234Odd-value increments of C<Y> indicate unstable ("development") releases that should not be used in production. C<Z> increments have no semantic meaning; they indicate only successive development releases.
235
236=back
237
238See the Changes file included with releases for an indication of the nature of
239changes involved.
